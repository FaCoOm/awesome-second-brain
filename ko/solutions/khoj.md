# Khoj

## 개요

- 웹사이트 / 문서: https://docs.khoj.dev/
- 회사 / 관리자: Khoj
- 상태: 활성 오픈소스 개인 AI
- 오픈소스: 예
- 배포: Khoj Cloud 또는 self-hosted
- 주요 사용자: 파일, 노트, 문서 위의 개인 AI를 원하는 사람
- 세컨드 브레인 역할: 기존 지식 소스 위의 개인 AI
- 마지막 검토: 2026-05-31

## 한 줄 요약

Khoj는 파일과 노트에 대해 대화하고, 자연어로 문서를 검색하며, cloud 또는 self-hosted 경로로 실행할 수 있는 오픈소스 개인 AI입니다.

## 세컨드 브레인 적합도

Khoj는 PDF, plaintext, Markdown, org-mode, Notion page, note를 Obsidian, Emacs, desktop, browser 같은 앱을 통해 질문하고 싶은 사용자에게 좋은 세컨드 브레인 선택지입니다.

## 기능 평가

| 영역 | 평가 |
|---|---|
| 배포 / 소유권 | Cloud 또는 self-hosted. |
| 맥락 수집 | PDF, plaintext, Markdown, org-mode, Notion page 등 file/note source 내장 지원. |
| 지식 정리 | 문서에 대한 내장 인덱싱/검색. 깊은 schema/graph curation이 주 모델은 아닙니다. |
| Memory 발전 | 부분 지원. retrieval과 assistant workflow는 유용하지만 이름 붙은 dream loop가 핵심 positioning은 아닙니다. |
| 검색 / 활용 | 공유 파일 위의 자연어 검색과 채팅. |
| 에이전트 활용 / 쓰기 반영 | app/client 우선: Emacs, Obsidian, desktop, browser. API/agent 세부사항은 대상 설정에서 확인해야 합니다. |
| 개인 / 팀 범위 | 배포/출처 선택으로 가능하지만 가장 잘 맞는 용도는 개인 AI입니다. |
| 검토 / 정정 | web/desktop/client interface와 source-specific workflow. |
| 프라이버시 / 통제권 | privacy를 위한 self-hosting option과 convenience를 위한 cloud option. |
| 설정 / 운영 | 중간. cloud는 더 빠르고 self-hosting에는 runtime/source setup이 필요합니다. |

## 강점

- "your second brain" positioning이 명확함.
- 개인 문서와 노트 Q&A에 강함.
- open-source와 self-host 가능.
- Obsidian/Emacs 사용자에게 좋은 bridge.

## 한계

- 많은 AI 도구 전반의 end-to-end 세컨드 브레인 계층이 주 초점은 아님.
- developer memory API나 hosted second-brain platform보다 write-back/shared team memory 초점은 약함.
- 출처 최신성과 self-hosting 운영은 사용자 책임.

## 잘 맞는 경우

- 파일, 노트, PDF, 문서 위의 개인 AI를 원하는 사람.
- Obsidian, Emacs, Markdown, self-hosting user.
- 넓은 second-brain automation보다 출처 Q&A가 중요한 workflow.

## 잘 맞지 않는 경우

- 많은 도구/정보원에 걸친 shared memory governance가 필요한 팀.
- 많은 AI 도구의 automatic write-back이 필요한 사용자.
- memory API가 primary product primitive인 builder.

## 트레이드오프

Khoj는 cloud/self-hosted 선택지와 함께 개인 문서/노트 검색에 최적화되어 있습니다. 트레이드오프는 많은 도구와 teammate에 걸친 low-friction end-to-end 세컨드 브레인보다는 출처 위의 개인 AI에 가깝다는 점입니다.
